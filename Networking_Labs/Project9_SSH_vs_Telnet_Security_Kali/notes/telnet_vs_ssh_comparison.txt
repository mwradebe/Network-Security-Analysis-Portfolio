========================================
COMPARATIVE ANALYSIS
========================================

TELNET WEAKNESSES:
1. Zero encryption (all data plaintext)
2. Passwords transmitted as typed
3. Commands and outputs fully readable
4. No protection against network sniffing
5. No protection against man-in-the-middle attacks

SSH STRENGTHS:
1. Full encryption of all data
2. Passwords never visible on network
3. Commands and outputs protected
4. Resistant to network sniffing
5. Protects against man-in-the-middle attacks
6. Supports key-based authentication (even stronger)

========================================
CONCLUSION
========================================

Through practical packet analysis, I have demonstrated
that Telnet transmits ALL data in plaintext, including
usernames and passwords. An attacker with network access
can trivially capture and extract credentials.

SSH encryption prevents this attack entirely. Even with
full packet captures, attackers cannot recover credentials
or read command data.

There is NO valid security justification for using Telnet
in 2025. The migration cost is minimal compared to the
massive security risk of credential exposure.

VERDICT: TELNET MUST BE ELIMINATED FROM ALL NETWORKS.
